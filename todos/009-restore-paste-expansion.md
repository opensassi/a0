### What Remains
**Implement `xExpandPastePlaceholders()` in `src/tui/agent_tui.cpp`**

The function is declared in `src/tui/agent_tui.h:98`:
```cpp
std::string xExpandPastePlaceholders(const std::string& input);
```

But has **no implementation** in `agent_tui.cpp`. It must:
1. Scan `input` for `[ PASTED #N ]` markers (where N is a positive integer)
2. Look up each N in `m_pastedContents` (an `unordered_map<int, string>`)
3. Replace each marker with the stored content
4. Return the expanded string
5. Purge used entries from `m_pastedContents` (keep unused ones so `[ PASTED #1 ]` can be typed manually)

**Wire it into `xHandleSubmit()`**

In `src/tui/agent_tui.cpp:342`, the `input` is sent directly:
```cpp
m_cmdSender.send(mpsc::SubmitGoal{input});
```
It should call expand first:
```cpp
std::string expanded = xExpandPastePlaceholders(input);
m_cmdSender.send(mpsc::SubmitGoal{expanded});
```
And use `expanded` for the user message entry + history instead of `input`.

```python
def _wait_for_text(self, driver, text, timeout=15):
    deadline = time.monotonic() + timeout
    while time.monotonic() < deadline:
        t = driver.capture(timeout=2)
        if text in t:
            return True
    return False
```

## Key Technical Details
- **`xExpandPastePlaceholders` signature**: `std::string (const std::string& input)`
- **`m_pastedContents` type**: `std::unordered_map<int, std::string>` (id â†’ stored content)
- **Placeholder format**: `[ PASTED #N ]` (with spaces around N, as generated by `xProcessPasteBuffer` at `agent_tui.cpp:567`)
- **Call site**: `xHandleSubmit()` at `agent_tui.cpp:350` â€” replace `input` with expanded form before `SubmitGoal` and history/UI
- **Edge case**: If `m_pastedContents` is empty, function should return input unchanged (no-op)
- **Edge case**: If marker is split across multiple paste operations (sequential `[ PASTED #1 ][ PASTED #2 ]`), both must be expanded
- **Edge case**: Manually typed `[ PASTED #N ]` with valid N should expand too (if N exists in `m_pastedContents`)
